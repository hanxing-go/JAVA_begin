## 注解
1. 注释(comment)：给人看的，编译后就没了，不参与程序逻辑
2. 注解(annotation)：给“程序/工具/框架"看的，能够被编译器处理、也可能被运行时反射读取。
用于修饰解释 包、类、方法、属性、构造器、局部变量等数据信息。

和注释一样，注解不影响程序逻辑，但注解可以被编译或运行，相当于嵌入代码中的补充信息。

### 三个基本的Annotation
1. @Override: 限定某个方法，是重写父类方法，该注解只能用于方法
```Java
class Father{
    public void fly(){
        sout("father fly");
    }
}

class Son{
    @Override
    public void fly() {
        sout("son fly");
        //如果写了@Override注解，编译器会检查方法是否真的重写了父类的方法，如果重写，则编译通过，如果没有则构成重写，则编译错误。
    }
}
```
2. @Deprecated: 用于表示某个程序元素（类，方法等）已过时
   可以修饰方法、类、字段，包，参数等等
   @ Deprecated 的作用可以做到新旧版本的兼容和过渡


3. @SuppressWarnings: 抑制编译器警告
    当并希望看见警告的时候，可以用SuppressWarnings来抑制警告信息
    ```java
    public void f1(){
        @SuppressWarnings({"rawtypes"})
        List list = new ArrayList();
    }
    ```
### JDK的元Annotation(元注解)
经常在源码能看见，JDK的元Annotation用于修饰其他Annotation
1. Retention //指定注解的作用范围
2. Target    //指定注解可以在哪些地方使用
3. Documented//指定该直接是否会在javadoc体现
4. Inherited //子类会继承父类注解
